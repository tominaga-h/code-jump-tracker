# VS Code拡張機能 要件定義書

**プロジェクト名**: Code Jump Tracker

## 1. 概要

VS Codeでのコードリーディングやコードレビュー中において、「定義へ移動」や「手動でのマーキング」を行った際の移動履歴を視覚的に管理する拡張機能。
シェルの `pushd / popd / dirs` のような直感的な履歴管理と、Webviewを用いた美しいテーブルUIを提供し、複雑なコードベースでの「迷子」を防ぐ。

## 2. 目的・課題

- **課題:** VS Code標準の「戻る」機能では履歴が視覚化されておらず、複数回のジャンプを行った後に目的の場所へ正確に戻るのが困難。
- **目的:** ジャンプ履歴と手動で記録した場所をサイドバーで一覧化し、自由に行ったり来たりできる環境を提供することで、開発者のコード把握とレビューの効率を劇的に向上させる。

## 3. 機能要件

### 3.1. 履歴の記録（Push）トリガー

以下の2つの方法で現在地を履歴（スタック）に追加する。

- **A. 自動記録（Auto Push）**
  - 「定義へ移動」「参照へ移動」「実装へ移動」の実行時。
  - アウトライン（シンボルツリー）からの移動時。
  - ファイル間の移動、または同一ファイル内での大きな行移動が発生した時。
  - ※1行ずつのカーソル移動など、細かな移動は記録から除外する。
- **B. 手動記録（Manual Push）**
  - エディタ内、または行番号（ガター）の右クリックメニュー（コンテキストメニュー）に「履歴に保存（Push to Tracker）」コマンドを追加。
  - キーボードショートカットからも実行可能にする。

### 3.2. サイドバー表示（Webview UI）

VS Codeのサイドバーに専用のパネルを追加し、Webview（HTML/CSS）を用いて以下の要件を満たすUIを描画する。

- **2つのビュー（タブ等で切り替え）**
  1. **Jump History Log:** 移動の軌跡をすべて残すログ形式（例: `a → b → c → b → d`）。
  2. **Unique Visited Locations:** 重複を排除した訪問場所の一覧（例: `a → b → c → d`）。
- **テーブル状のレイアウト（列揃え）**
  - Flexbox等を用いて、各項目が縦に綺麗に揃うように描画する。
  - 表示フォーマット: `[手動アイコン] [ファイル名] [行番号] [シンボル名(関数/メソッド等)]`
  - 手動で追加された履歴には、先頭に「👉」などのアイコンを表示し、自動記録と視覚的に区別する。

### 3.3. エディタ上の視覚マーカー（Gutter Icon）

- 手動で履歴に保存した行には、エディタの行番号の横（ガター部分）にピン（📌）などのマーカーアイコンを永続的に表示する。
- これにより、サイドバーを見なくても「意図的にマークした場所」であることを視覚的に把握できる。

### 3.4. 履歴の永続化とスコープ

- 履歴は**ウィンドウ（ワークスペース）ごと**に独立して管理する。
- VS Codeの `workspaceState` API を利用し、エディタを再起動しても前回の履歴とガターアイコンの表示を完全に復元する。

### 3.5. 履歴の管理・クリア

- **個別削除:** サイドバーの各履歴アイテムに「ゴミ箱アイコン（または×ボタン）」を配置し、個別に削除可能にする。
- **全クリア:** パネル上部に「全クリアボタン」を配置し、履歴をリセット可能にする。
- **自動クリーンアップ:** 保存上限数（後述）を超えた場合、最も古い履歴から自動的に削除（Pop）される。

### 3.6. キーボードショートカット

サイドバーのクリックだけでなく、キーボード操作で履歴を移動・操作できるコマンドを提供する。（以下はデフォルト案）

- **前の履歴に戻る:** `Alt + Shift + ←` (Win) / `Ctrl + Shift + -` (Mac)
- **次の履歴に進む:** `Alt + Shift + →` (Win) / `Ctrl + Shift + =` (Mac)
- **手動で履歴に保存:** 任意のショートカットを割り当て可能にする。

## 4. ユーザー設定（カスタマイズ）

VS Codeの `settings.json`（設定画面）から、以下の項目を変更できるようにする。

- `codeJumpTracker.maxHistorySize`: 履歴の保存上限数（デフォルト: 50）

## 5. 非機能要件

- **パフォーマンス:** カーソル移動のたびに重い処理が走らないよう、デバウンス（遅延実行）などを取り入れ、VS Codeの動作を重くしない設計にする。
- **OS互換性:** Windows, macOS, Linux環境で正常に動作すること。
